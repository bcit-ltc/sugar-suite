Experimental Loader

Description

Automatically loads experimental CSS and JavaScript files when the body element has the "experimental" class. This allows for easy activation of experimental features across an entire page.

Usage

Add the experimental class to the <body> element to automatically load experimental features.

Available Options

  • CSS Classes:
    • body.experimental - Triggers automatic loading of experimental files
  • Automatic Features:
    • Loads ../css/experimental.css stylesheet
    • Loads ../js/experimental.js JavaScript file
    • Provides console feedback on successful/failed loading

Examples

Basic Experimental Page

To enable experimental features on a page, add the experimental class to the body:

<!DOCTYPE html>
<html lang="en" class="no-js">
<head>
    <meta charset="utf-8">
    <title>My Experimental Page</title>
    <link rel="stylesheet" href="../css/bcit.css">
</head>
<body class="experimental">
    <div class="container">
        <h1>Experimental Features Enabled</h1>
        <p>This page will automatically load experimental CSS and JavaScript.</p>
    </div>
    <script src="../js/lat.js"></script>
</body>
</html>

Conditional Loading

The experimental loader only activates when the body has the experimental class:

<!-- This page will NOT load experimental files -->
<body>
    <!-- Standard features only -->
</body>

<!-- This page WILL load experimental files -->
<body class="experimental">
    <!-- Experimental features available -->
</body>

File Loading Process

When experimental mode is enabled, the loader:

  1. Detects the body.experimental class
  2. Dynamically creates and loads the experimental CSS file
  3. Dynamically creates and loads the experimental JavaScript file
  4. Provides console feedback on loading success/failure

Notes

  • Experimental features are loaded dynamically and asynchronously
  • Files are loaded from relative paths: ../css/experimental.css and ../js/experimental.js
  • Loading success/failure is logged to the browser console
  • Experimental features are separate from core features
  • Use this for testing new features before they become core
  • Experimental files should be included in the build process
  • Consider performance implications of loading additional files
  • Test experimental features across different browsers
  • Document experimental features separately from core features
