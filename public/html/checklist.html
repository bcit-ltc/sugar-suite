Checklist

Description

Creates interactive checklists with persistent storage, hierarchical checking, and optional strikethrough effects. Perfect for task lists, course progress tracking, and interactive learning activities.

Usage

Wrap a list (ul or ol) in a container with class .checklist. Each list item becomes a checkable item. Use special syntax for conditional content display.

Available Options

  • CSS Classes:
    • .checklist - Main container
    • .strikethrough - Adds strikethrough effect to checked items
    • .hide-after - Hides content after checking (on list items)
    • .show-after - Shows content after checking (on list items)
  • Special Syntax:
    • @ - Prefix for conditional content that shows after checking

Notes

  • Checklist state is automatically saved to browser's localStorage
  • Parent items are automatically checked when all children are checked
  • Use @ at the beginning of text to create conditional content
  • .hide-after hides content when item is checked
  • .show-after shows content when item is checked
  • .strikethrough adds strikethrough effect to checked items
  • Supports nested lists with automatic hierarchical checking
  • Each checklist gets a unique ID based on the page URL
  • Content is preserved across browser sessions
