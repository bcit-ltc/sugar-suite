Line Matching

Description

Creates interactive matching exercises where users connect questions to answers by drawing lines. Supports one-to-one and one-to-many matching with visual feedback and automatic shuffling.

Usage

Wrap a table with class .line-matching containing a figure.table element. Each row represents a question-answer pair. The table is automatically converted into an interactive matching interface.

Available Options

  • CSS Classes:
    • .line-matching - Main container (required)
    • .left-order - Keeps questions in original order, shuffles answers
  • Structure:
    • <figure class="table"> - Required wrapper for the data table
    • <thead> - Optional headers for questions and answers
    • <tbody> - Contains the question-answer pairs

Notes

  • Click on a question to start drawing a line, then click on the corresponding answer
  • Lines are drawn dynamically as you move the mouse
  • Correct matches create permanent lines, incorrect matches reset the selection
  • One-to-many matching is supported by repeating the same question with different answers
  • Use .left-order to keep questions in original order while shuffling answers
  • The Reset button clears all matches and allows starting over
  • On mobile devices, lines are hidden during interaction for better usability
  • Supports keyboard navigation (Enter key to select items)
  • Items are automatically shuffled unless using .left-order
  • Can include images, text, or mixed content in matching items
